## 1. V1.1 用户端目标

V1.1 用户端的目标不是新增一个泛泛的“展示端”，而是把营地现场真实运营链路打通：

1. 老师在每日任务中记录个人、队伍、项目、特殊情况和路演材料。
2. 记录确认后，学员和家长在对应工作台看到可读、可追溯的每日记录。
3. 家长可对单条记录填写回执，形成家校沟通闭环。
4. 注册建档补齐营前所需信息，减少开营前人工收集成本。

## 4. 老师端 V1.1 运营需求

### 4.1 我的队伍

对应页面：`06-teacher-dashboard-teams.html`

需求规则：

- 所有老师工作台页面增加班级选择。
- 展示老师负责的队伍、队员、项目方向和进展。
- 本期去掉“队伍点评 / 项目点评”入口。
- 历史的队伍点评、项目点评弹窗暂存，不作为本期入口。

### 4.2 学生管理

对应页面：`07-teacher-dashboard-students.html`

最新范围：

- 页面定位从“学生点评”改为“学生管理”。
- 列表主要查看学生档案、所属队伍、项目、关注状态。
- 操作只保留“查看信息”。
- 查看信息弹窗内可修改关注状态和关注备注。
- 本期不做“写寄语”“查看记录”。

### 4.3 每日任务与记录

对应页面：`08-teacher-dashboard-tasks.html`

记录类型：

| 类型 | 需求 |
| --- | --- |
| 个人记录 | 每天都有；记录行为表现、能力观察、情绪/状态、老师建议。 |
| 队伍记录 | 每天都有；记录队伍概况、协作观察、冲突/卡点、老师建议。 |
| 项目记录 | 每天都有；与 Workshop / milestone 关联，记录项目进展、关键结果、下一步。 |
| 特殊情况 | 情况类型支持多选，用于健康、安全、情绪、纪律等事件。 |
| 路演材料 | Day5 / Day6 出现；Day5 只保留 PPT、宣传片，产品原型选填。 |
| 每日汇总 | 晚间总结推送管理员，用于运营复盘。 |

命名规则：项目点评 / 队伍点评统一改名为项目记录 / 队伍记录。

### 4.4 记录推送

对应页面：`09-teacher-dashboard-reports.html`

规则：

- 每条记录确认后才推送。
- 未推送记录可进入修改。
- 推送后不可修改，如需调整应走撤回/重新发布流程。
- 晚间总结推送管理员。
- 个人 / 队伍 / 项目记录分别显示绑定来源、最后保存时间、推送范围。

## 5. 学员端每日记录需求

对应页面：`22-student-dashboard-day-flow.html`

### 5.1 页面结构

- 顶部展示学员、营地、当前 Day、队伍和项目方向。
- 每日记录按 Day1-Day6 展示。
- 每天都有个人记录、队伍记录、项目记录。
- Day5 / Day6 增加项目资料类记录。
- 记录卡片支持横向滑动查看超出范围的信息。

### 5.2 详情弹窗字段映射

学生端详情弹窗必须和老师端记录表一一对应：

| 记录类型 | 展示字段 |
| --- | --- |
| 个人记录 | 行为表现、能力观察、情绪/状态、老师建议 |
| 队伍记录 | 队伍概况、协作观察、冲突/卡点、老师建议 |
| 项目记录 | 项目概况、Milestone 目标拆解、关键结果、产物和过程照片、目标达成评估、下一步计划、带队老师建议 |
| 项目材料类记录 | 项目概况、材料清单、提交/推送状态、建议 |

### 5.3 详情摘要收窄

详情卡片不再显示以下 4 项：记录类型、绑定来源、当前状态、可见对象。学员端摘要只保留发布时间、阶段。

## 6. 家长端每日记录需求

对应页面：`23-parent-dashboard-day-flow.html`

### 6.1 页面结构

- 支持多孩子切换。
- 去掉“工作成果展示”，每日记录成为核心信息区。
- 每日记录数据和学员端对齐：Day1-Day6 每天都有个人记录、队伍记录、项目记录。
- Day3/Day4 项目记录绑定 Workshop 3-6 milestone。
- Day5 增加路演资料：PPT、宣传视频、产品说明书、商业模式、逐字稿、2 次彩排 milestone。
- Day6 增加最终材料上传、材料推送家长。

### 6.2 详情弹窗

- 家长端详情弹窗复用老师记录字段，但文案转成家长可读表达。
- 摘要保留：孩子、队伍、项目方向、发布时间、阶段。
- 不显示：记录类型、绑定来源、当前状态、可见对象。

### 6.3 家长回执

- 家长打开任意记录详情后，可以输入一段回执。
- 点击“提交回执”后，弹窗内显示已回执卡片和回执文字。
- 每条记录各自保存一份回执，不能串到其他记录。
- 后续正式实现需要后端保存回执、记录已读时间和操作者。

### 6.4 手机端顺序和滑动

手机端/窄屏顺序按已确认规则：

1. 顶部导航
2. 家长信息
3. 选择孩子
4. 筛选
5. 学员队伍和项目信息
6. 每日记录
7. 结营报告
8. 我的课程
9. 营地相册
10. 问题反馈

每日记录区域需要横向滑块或按钮，超出范围的信息可滑动查看。

## 8. 验收标准

- 老师端每日任务里不同记录类型能结构化保存。
- 学员/家长详情弹窗和老师记录字段一一对应。
- 家长端去掉工作成果展示后，每日记录数据完整可滑动。
- 家长可对单条记录提交回执。
- 学生管理页只保留查看信息和关注状态维护，不出现本期外入口。
