## 1. V1.1 定位

### 1.1 版本目标

V1.1 聚焦“把已经具备基础模块的后台，变成营地工作人员可连续使用的运营工具”。本版本的六个交付重点：

1. **后台体验优化**：减少跨页面、跨模块反复选择营地和人工核对的成本。
2. **老师点评系统**：让老师可以在权限范围内记录事实、完成点评、给出评分，并形成可审核的对外内容。
3. **轻量 AI 点评美化**：通过“提示词模板 + 受控 AI 接口”润色老师已写的点评，不改变事实，不自动发布。
4. **招生信息收集优化**：将现有外部问卷链接管理升级为可配置、可追踪、可审核、可导出的营前数据流程。
5. **营地组织数据重构**：明确用户、营地成员、班级、队伍和老师职责之间的单一数据来源，支持批量建队与分期管理。
6. **内容与第三方课程权限中心**：同步小鹅通课程、关联营地与学习对象，并由本系统控制课程权益。

### 1.2 V1.1 不做什么

- 不交付完整 AI 成长报告、AI 内容生成和 AI 表单智能体；这些归入 V1.4。
- 不交付顾问老师和完整的投资人评审业务；这些归入 V1.2。V1.1 只交付投资人评委账号身份、邀请、审核及最小数据边界。
- 不交付积分、徽章、排行榜和权益兑换；这些归入 V1.3。
- 不以“已有客户端设计图”作为 V1.1 的默认开发范围。老师高频动作的最终载体（后台响应式页面或独立 H5）需结合原型确认。

### 1.3 V1.0 依赖基线

V1.1 在已上线后台能力之上迭代：系统用户、营地、班级、营员、老师、队伍、课程、Workshop、任务、报名审核、问卷链接、素材、营地相册、家长反馈、学生 Idea。

## 3. 共用业务规则

### 3.1 当前营地上下文

- 后台业务页面必须显示当前营地；页面内数据默认受当前营地过滤。
- 若用户有多个营地权限，可在顶部切换；切换后刷新班级、队伍、人员、内容与数据统计。
- 无营地权限或营地已归档时，显示明确空态，不允许误写入其他营地数据。

### 3.2 老师权限范围

| 老师身份 | 可查看范围 | 可记录/点评范围 | 不可操作 |
| --- | --- | --- | --- |
| 带队老师 | 负责队伍学员 | 负责队伍学员 | 非负责队伍、系统配置、未授权隐私字段 |
| 生活老师 | 当前营地全部学员的必要基础与安全信息 | 当前营地生活/自理类记录 | 发布成长结论、修改系统权限 |
| 驻场导师 | 所带班级学员 | 所带班级的学习/项目记录与点评 | 非所带班级信息 |
| 定义管理员 | 授权营地全部信息 | 审核、发布、配置模板和导出 | 超级管理员系统级能力 |

### 3.3 点评内容分层

| 层级 | 用途 | 可见人 |
| --- | --- | --- |
| 原始记录 | 老师留存事实、关键事件、介入方式、证据 | 授权老师、管理员 |
| 内部点评 | 教师协作、待审核的结论或建议 | 授权老师、管理员 |
| 对外点评 | 适合学员/家长阅读的已审核内容 | 学员、已绑定家长 |

### 3.4 轻量 AI 美化边界

1. AI 输入仅包含老师主动选择的原始点评文本、适用提示词模板和必要的称谓信息。
2. AI 只做语气、结构、表达清晰度和鼓励性建议的润色，**不得新增事件、成绩、诊断或未经老师确认的判断**。
3. AI 返回内容必须与原文并列显示；老师可以编辑、重新生成、放弃或确认。
4. 确认后保存原文、美化稿、模板版本、调用时间、调用人及最终采纳状态。
5. 所有对家长/学员可见的内容仍遵循人工审核与发布流程。

### 3.5 用户、营地成员、班级和队伍的数据边界

1. **用户管理**是账号和基础身份的唯一来源，维护真实姓名、账号、手机号、出生日期、账号类型与教师类型；不得将营地、班级、队伍直接固化在用户资料中。
2. **营地成员关系**是“某个学员参加某一期营地”的唯一来源。同一学员在同一营地只能有一条有效关系，但可参加不同期营地。
3. 班级、队伍、队内身份均挂在营地成员关系上；队伍详情的队员、队长和成员数量从该关系实时汇总，不能重复维护姓名。
4. 老师的账号身份和教师类型在用户管理中定义；老师的营地、班级、队伍、项目职责通过独立授权关系配置。

### 3.6 营地状态的管理员确认原则

- 营地状态统一为：`未开营`、`招生中`、`已招满`、`开营中`、`已结营`。
- 系统根据日期、有效报名人数与容量给出切换提醒，但状态变更必须由管理员确认，以处理延期、取消、扩容等真实运营情况。
- 当前报名人数只统计“有效报名且未取消”的学员；最大容量由管理员配置。

## REQ-101 后台信息架构与全局营地上下文优化

**模块**：后台体验｜**优先级**：P0｜**状态**：待开发

### 用户与目标

- 运营/教务：快速确认自己正在管理哪个营地，避免在人员、内容和任务模块中误操作。
- 定义管理员：在多营地之间切换并清晰识别不同营期的数据边界。

### 功能要求

1. 在营地业务后台提供全局营地选择器，显示营地名称、期次、状态与日期。
2. 班级、队伍、营员、老师、Workshop、任务、报名、相册等页面默认继承当前营地。
3. 页面标题、面包屑和导出文件名包含当前营地/期次标识。
4. 无当前营地时，禁止创建依赖营地的对象，并引导选择或创建营地。

### 验收

- 切换营地后，列表、筛选项和新增表单不出现上一营地的可选对象。
- 用户刷新或返回页面时，系统按约定保留当前营地上下文。
- 所有查询仍由服务端进行营地与权限校验，不能仅依赖前端过滤。

## REQ-102 营地配置向导与完成度检查

**模块**：后台体验｜**优先级**：P0｜**状态**：待开发

### 功能要求

1. 为营地提供配置检查清单：营地信息、班级、老师、学员、家长关系、队伍、课程/Workshop、任务、招生表单、相册/素材。
2. 每项显示未开始、进行中、已完成、存在异常四种状态，并可跳转到处理页面。
3. 在营期开始前提示关键缺失项，例如无营员、无负责老师、无任务、关键信息未完成。

### 验收

- 配置状态应由真实数据自动计算，不允许仅人工勾选完成。
- 每个异常项至少提供问题说明和可执行入口。

| 字段组 | 建议字段 |

## REQ-106 任务点评、维度评分与发布审核

**模块**：老师点评｜**优先级**：P0｜**状态**：待开发

1. 老师可在任务提交详情中查看提交版本、附件、提交时间及逾期状态。
2. 点评支持点评模板、结构化亮点/建议、星级或维度评分、内部备注和对外可见范围。

- 任务截止前学员可修改；老师点评应绑定具体提交版本。
- 截止后、补交、撤回、重新点评等状态变化可追溯。
- 评分汇总可用于营期运营查看，但不自动生成对外成长结论。

## REQ-107 AI 点评美化（提示词模板 + AI 接口）

**模块**：老师点评｜**优先级**：P0｜**状态**：待开发

### 使用流程

1. 老师完成原始点评或选择已有点评草稿。
2. 选择美化模板，例如“鼓励且具体”“面向家长的成长反馈”“任务完成点评”。
3. 系统在服务端组合模板与老师原文，调用受控 AI 接口。
4. 页面展示原文、美化结果与差异；老师可以编辑、重新生成、放弃或确认。
5. 确认后保存最终稿；若提交对外发布，仍进入人工审核流程。

### 模板管理

| 字段 | 说明 |
| --- | --- |
| 模板名称 | 供老师选择的短名称 |
| 适用场景 | 任务点评、营地记录、家长沟通等 |
| 提示词正文 | 后台维护，仅授权管理员可编辑 |
| 语气 | 鼓励、客观、具体、适龄等约束 |
| 禁止项 | 不编造事实、不贴标签、不做诊断、不承诺结果 |
| 版本号 | 每次修改生成新版本，历史调用保留旧版本 |
| 启用状态 | 控制模板是否可被老师使用 |

### 接口与数据要求

- AI 接口只允许服务端调用；前端不保存密钥。
- 发送前对身份证、地址、联系电话、健康等敏感字段进行剔除或脱敏。
- 请求记录：操作者、点评 ID、模板版本、时间、模型/服务标识、成功/失败状态；不在日志中持久化不必要的完整敏感文本。
- 设置超时、失败提示和人工继续编辑的降级路径。

### 验收

- 美化结果不能直接覆盖原文；原文始终可回看。
- 每条最终点评可识别是否使用 AI、使用何种模板版本、老师是否编辑过。
- 不提供“自动发布”“自动生成事实”“一次性批量替换全部点评”的能力。

### 原型与补充区

- 原型链接/图片：`待补充`

- AI 服务商与接口契约：`待确认`

## REQ-108 招生信息收集表单配置与版本管理

**模块**：招生信息｜**优先级**：P0｜**状态**：待开发

### 功能要求

1. 统一管理成长档案、营地信息、入营评测等表单；首期可迁移/承接已有外部问卷链接。
2. 支持字段类型：单行文本、多行文本、单选、多选、日期、上传、说明、确认项。
3. 每个字段可配置：标题、说明、必填、选项、校验规则、是否敏感、可见角色、导出列名。
4. 表单发布后生成版本；新版本不应覆盖已提交历史答案。
5. 支持草稿、提交、退回补充、锁定和停用状态。

### 验收

- 可按营地、报名阶段、学员状态配置使用的表单版本。
- 运营人员可查看字段级缺失、整体完成率和异常数据。

## REQ-114 营地五态管理、招生容量与状态流转

**模块**：营地管理｜**优先级**：P0｜**状态**：待开发｜**建议排期**：第 1-3 周

### 状态定义

| 状态 | 标识建议 | 核心含义 | 主要行为 |
| --- | --- | --- | --- |
| 未开营 | 灰色 | 已创建，尚未开始招生 | 可编辑营地和配置内容，不对外报名 |
| 招生中 | 蓝色 | 正在接受报名，尚未满额 | 开放报名，展示报名人数与剩余名额 |
| 已招满 | 橙色 | 有效报名人数达到容量 | 关闭普通报名，可扩容或处理候补 |
| 开营中 | 绿色 | 营期正式开始 | 停止常规报名，开放记录、任务、相册等营中能力 |
| 已结营 | 红色/深灰 | 营期结束 | 停止日常记录/任务提交，保留报告、导出、复盘与历史查看 |

### 页面与规则

1. 营地列表字段调整为：营地名称、营地状态、营期时间、报名进度、年龄范围、营地描述、操作。报名进度格式示例：`86 / 150（余 64）`、`150 / 150（已满）`。
2. 状态筛选统一为：全部、未开营、招生中、已招满、开营中、已结营。
3. 营地详情统计卡按状态显示动态信息：招生中显示报名人数、容量、剩余名额、距开营/报名结束；已招满显示招满时间、候补人数、扩容状态；开营中显示营期天数、任务完成率、教师记录完成率；已结营显示报告生成率、发布率与可导出数据。
4. 状态流转：未开营可由管理员“开始招生”进入招生中；达到容量时系统提示切换已招满，但管理员确认后生效；招生中/已招满到开营日时提示切换开营中；已招满扩容或有人退出后可返回招生中；开营中由管理员确认结营。
5. 不完全依赖日期自动切换。系统只提示“今日应开营/已满额”，管理员确认才改变状态；每次变更保留操作人、时间、原因。

### 状态权限限制

| 状态 | 管理员 | 学员/家长 | 教师 |
| --- | --- | --- | --- |
| 未开营 | 编辑、配置内容、开始招生 | 不报名、不进入营地内容 | 查看已授权准备资料 |
| 招生中 | 审核报名、导入学员、调整容量 | 报名、填写营前信息 | 查看已分配学员 |
| 已招满 | 候补、扩容、调整名单 | 普通报名关闭 | 查看已分配学员 |
| 开营中 | 管理人员、任务、相册、运营数据 | 学习、查看课程、提交任务 | 填写记录、点评任务 |
| 已结营 | 审核报告、导出、归档 | 查看已发布内容和报告 | 完善点评、生成/确认报告 |

3. 新增营地课程从已同步的小鹅通课程中选择，配置营地、展示名称、学习起止时间、学习对象（全体学员/指定班级/指定队伍/指定学员）、教师权限、家长权限和发布状态。
4. 默认权益：学员可学习；带队和驻场老师可按需查看；生活老师默认无课程权限；家长默认仅看课程介绍和学习进度，不默认进入完整课程。
5. 用户进入课程页时，系统先校验是否在当前营地课程学习对象内，再查询第三方账号关联；未绑定时创建或绑定第三方身份并发放权益，然后跳转或嵌入课程页。

### 第三方授权与数据边界

1. 管理员配置小鹅通 App ID、应用密钥和 IP 白名单；密钥保存后仅显示脱敏值。提供前往授权、测试连接、手动同步入口，并记录最近同步结果。
2. 新增第三方账号关联对象：系统用户 UUID、平台类型、小鹅通用户 ID、OpenID（若返回）、脱敏手机号、绑定状态、最近授权时间。
3. 手机号只可作为首次匹配线索，首次绑定后必须保存第三方唯一用户 ID，不可仅凭手机号直接合并账号。
4. 课程详情提供课程信息、营地授权、学习数据三个页签。学习数据同步、H5 免重复登录等能力必须以当前店铺实际开通的开放接口权限为准，不能作为默认承诺。
5. 小鹅通接入需完成应用授权、`access_token`、接口权限和 IP 白名单配置；具体接口在技术评审时逐项验收。[小鹅通开放 API 概述](https://api-doc.xiaoe-tech.com/)｜[接入步骤](https://api-doc.xiaoe-tech.com/read_before/access_guide.html)
