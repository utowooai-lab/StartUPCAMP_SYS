# 少年马斯克创业营营地管理系统产品需求梳理

版本：v0.1  
日期：2026-05-29  
梳理角度：用户使用、技术开发、产品运营、交互设计  
适用对象：产品评审、研发排期、运营配置、设计重构

## 1. 产品定位

少年马斯克创业营营地管理系统是一个面向创业营业务的多角色营地管理平台。它不是单一的课程系统，也不是单一的后台管理系统，而是围绕一个营期，把学员、老师、家长、运营后台连接起来，支撑营前报名建档、营中课程任务与团队协作、营后点评沉淀和家长反馈。

一句话定义：

> 一个服务创业营全流程的「用户端工作台 + 后台运营管理系统」。

产品由两部分组成：

- 用户端：老师端、学员端、家长端。
- 后台管理端：营地、人员、课程、资料、任务、队伍、点评、相册、反馈等业务数据配置与管理。

## 2. 业务主线

系统应围绕四个阶段展开，而不是只围绕页面展开。

| 阶段 | 业务目标 | 主要用户 | 核心动作 |
| --- | --- | --- | --- |
| 营前 | 完成报名、账号、资料与入营准备 | 家长、学员、运营 | 注册、补充资料、填写表单、入营评测 |
| 营中 | 支撑学习、任务、工作坊和团队协作 | 学员、老师、运营 | 看课程、下载资料、提交任务、管理团队、提交创业想法 |
| 营后 | 沉淀成果、评价和回顾 | 老师、家长、学员 | 老师点评、任务点评、查看相册、查看课程资料 |
| 持续运营 | 管理数据、处理反馈、复用营地配置 | 运营、管理员 | 配置营地、维护用户、处理反馈、管理内容 |

## 3. 从用户使用角度梳理

### 3.1 用户角色与核心诉求

| 角色 | 使用场景 | 核心诉求 | 产品应解决的问题 |
| --- | --- | --- | --- |
| 学员 | 参加创业营期间 | 知道今天要做什么、资料在哪里、任务怎么交、团队是谁 | 减少信息分散，让学员能围绕营地任务行动 |
| 老师 | 带队和点评学生 | 快速找到学生、管理队伍、查看状态、提交点评 | 提升带队管理效率，沉淀老师评价 |
| 家长 | 关注孩子营地表现 | 查看课程、点评、照片，反馈问题 | 让家长有参与感和信任感 |
| 运营/管理员 | 配置和维护营地 | 管理营地、人员、课程、任务、资料和反馈 | 让营地可复制、数据可管理、过程可追踪 |

### 3.2 学员端使用路径

学员进入系统后，最重要的是完成“营地任务”和“创业想法提交”。

主要路径：

1. 登录或通过家长配置的学生账号进入系统。
2. 进入当前营地首页。
3. 查看个人身份：队长或队员。
4. 查看我的课程。
5. 下载相关资料或工作坊工具。
6. 查看营地任务。
7. 提交任务：上传图片、填写回答、提交。
8. 提交创业想法：填写标题、领域、想法描述、目标市场、竞品分析，上传商业计划书。
9. 查看我的团队和营地相册。

学员端的关键体验要求：

- 首页必须让学员一眼知道：课程、资料、任务、团队、相册在哪里。
- 队长与队员权限要明确区分，队长可以编辑队伍，队员只能查看。
- 任务提交入口要明显，提交后需要有状态反馈。
- 创业想法表单较长，需要分区清晰，避免学生不知道怎么填写。

### 3.3 老师端使用路径

老师端核心是“查学生、管队伍、做点评”。

主要路径：

1. 老师登录。
2. 进入老师营地页。
3. 通过姓名搜索学生，或按营地、学员筛选。
4. 查看学生列表和学生详情。
5. 在管辖队伍设置中选择负责队伍。
6. 在队伍成员管理中添加或移除队员。
7. 在队员点评中选择学员，填写点评内容和评分。
8. 设置点评可见性：公开或私有。

老师端的关键体验要求：

- 学生列表必须支持搜索、筛选、分页。
- 队伍管理要避免误操作，移除队员需要二次确认。
- 点评表单要适合老师快速填写，最好支持默认评分、常用评语或草稿。
- 老师只应看到自己负责营地、队伍或学员范围内的数据。

### 3.4 家长端使用路径

家长端核心是“看孩子表现”和“反馈问题”。

主要路径：

1. 家长登录。
2. 选择营地和孩子账号。
3. 查看孩子课程。
4. 查看老师点评和任务点评。
5. 查看云相册/营地照片。
6. 如有疑问，提交问题反馈。
7. 在设置中维护个人资料、学生账号、密码。

家长端的关键体验要求：

- 多孩子、多营地场景要支持切换。
- 点评内容要按老师点评、任务点评清楚分组。
- 相册要有良好的预览体验。
- 反馈要简单，支持图片、联系方式、匿名提交。

### 3.5 后台用户使用路径

后台用户核心是“先配置，再运营，再处理”。

主要路径：

1. 创建或维护营地/期次。
2. 创建用户账号，配置老师、学员、家长角色。
3. 绑定家长与学生账号。
4. 配置课程、资料、工作坊、任务。
5. 配置队伍、队长、队员、负责老师。
6. 查看任务提交、创业想法、点评。
7. 管理相册和反馈。
8. 跟踪报名资料、入营评测和营地状态。

后台体验要求：

- 数据配置要有清晰顺序，避免运营不知道先配什么。
- 需要支持按营地/期次隔离数据。
- 所有用户端展示内容都应能追溯到后台配置。
- 关键业务数据应支持导出，例如学员名单、任务提交、反馈列表。

## 4. 从技术开发角度梳理

### 4.1 系统架构建议

建议按“用户端门户 + 管理后台 + API 服务 + 文件服务 + 权限系统”来拆。

| 模块 | 说明 |
| --- | --- |
| 用户端门户 | 老师、学员、家长共用一套前端工程，通过角色渲染不同工作台 |
| 管理后台 | 运营配置、数据管理、权限管理 |
| API 服务 | 提供认证、营地、用户、课程、任务、团队、点评等接口 |
| 文件服务 | 管理图片、资料、计划书、任务附件 |
| 权限系统 | 控制角色、营地、队伍、学生绑定关系的数据访问 |

### 4.2 核心数据模型

技术开发不能只按页面建表，应先建立业务对象关系。

| 对象 | 关键字段 | 关系 |
| --- | --- | --- |
| User 用户 | id、姓名、手机号、头像、状态 | 可拥有一个或多个角色 |
| Role 角色 | 学员、老师、家长、管理员 | 决定功能权限 |
| Student 学员 | userId、姓名、性别、学校、身份证、家庭地址 | 关联家长、营地、队伍 |
| ParentBinding 家长绑定 | parentUserId、studentId、状态 | 支持一个家长绑定多个孩子 |
| TeacherProfile 老师档案 | userId、老师类型、负责范围 | 关联营地和队伍 |
| Camp 营地 | 名称、期次、封面、开始/结束时间、状态 | 承载课程、资料、任务等 |
| Course 课程 | 标题、内容、附件、排序、状态 | 属于某个营地 |
| Material 资料 | 名称、文件、可见角色、排序 | 属于某个营地 |
| Workshop 工作坊 | 标题、说明、工具附件 | 属于某个营地 |
| Task 营地任务 | 标题、描述、提交要求、截止时间 | 属于某个营地 |
| TaskSubmission 任务提交 | taskId、studentId、文本、图片、状态 | 可被老师点评 |
| Team 队伍 | 名称、项目名称、营地、队长 | 包含多个队员 |
| TeamMember 队员 | teamId、studentId、角色 | 区分队长/队员 |
| StartupIdea 创业想法 | 标题、领域、描述、目标市场、竞品分析、计划书 | 由学员提交 |
| Comment 点评 | 类型、内容、评分、可见性、点评人、被点评人 | 老师创建，家长/学员查看 |
| Album 相册 | 营地、图片、分组、上传人 | 家长/学员查看 |
| Feedback 反馈 | 类型、标题、描述、图片、联系方式、匿名、处理状态 | 家长或用户提交，后台处理 |

### 4.3 权限与数据隔离

权限是这个产品的技术重点。

| 场景 | 权限规则 |
| --- | --- |
| 学员看数据 | 只能看自己所属营地、自己的任务提交、自己的团队信息 |
| 队长编辑队伍 | 只能编辑自己队伍，不能跨队伍操作 |
| 老师看学生 | 只能看自己负责营地/队伍/学员范围 |
| 老师点评 | 只能点评负责范围内的学生 |
| 家长看孩子 | 只能看已绑定孩子的数据 |
| 管理员管理数据 | 按租户、营地、角色权限控制 |
| 文件访问 | 文件下载必须鉴权，不能只依赖公开 URL |

### 4.4 API 能力拆分

建议开发时按业务域拆 API，而不是按页面拆。

认证与账号：

- 登录、退出、验证码、注册、修改密码、个人资料修改。

营地：

- 营地列表、当前营地、营地详情、营地报名/绑定关系。

课程与资料：

- 课程列表、课程详情、资料列表、资料下载。

工作坊与任务：

- 工作坊列表、工具下载、任务列表、任务详情、任务提交、任务提交记录。

团队：

- 我的团队、编辑队伍、添加队员、移除队员、队伍列表、成员列表。

点评：

- 老师提交点评、点评列表、老师点评、任务点评、可见性控制。

创业想法：

- 提交创业想法、查看提交记录、后台审核或查看。

相册：

- 相册列表、照片列表、上传照片、查看照片。

反馈：

- 提交反馈、反馈列表、处理反馈、反馈状态更新。

### 4.5 开发风险

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| 角色关系混乱 | 家长、学生、老师、管理员共用账号体系，容易混乱 | 先设计清楚 User、Role、Profile、Binding |
| 数据越权 | 家长看到非绑定孩子、老师看到非负责学生 | 所有查询接口必须做服务端权限过滤 |
| 文件泄露 | 资料、计划书、任务图片可能包含隐私 | 文件访问使用鉴权或短期签名链接 |
| 页面驱动建模 | 只照 UI 做功能会遗漏后台数据关系 | 先完成数据模型和权限矩阵 |
| 多营地切换 | 用户可能关联多个营地 | 所有核心数据都要带 campId |
| 状态流转缺失 | 任务、反馈、资料填写需要状态 | 设计统一状态字段和操作日志 |

## 5. 从产品运营角度梳理

### 5.1 运营后台要解决什么

运营不是只“看数据”，而是要把一个营地从 0 配出来，并在营期中持续管理。

后台运营目标：

- 快速搭建一个新营地。
- 批量导入或创建老师、学员、家长。
- 配置课程、资料、任务、工作坊。
- 配置队伍和老师负责范围。
- 查看学员提交内容和老师点评。
- 管理相册和家长反馈。
- 复盘营地数据。

### 5.2 运营配置顺序

建议后台按这个顺序引导运营人员配置：

1. 创建营地/期次。
2. 配置报名与资料填写要求。
3. 导入或创建人员账号。
4. 绑定家长和学员。
5. 配置老师负责范围。
6. 创建队伍并分配队长/队员。
7. 发布课程、资料、工作坊。
8. 发布营地任务。
9. 营期中查看提交与点评。
10. 营期后导出数据、沉淀相册和成果。

### 5.3 运营指标

产品后续应支持以下数据看板：

| 指标 | 运营意义 |
| --- | --- |
| 注册人数 | 判断报名转化 |
| 资料填写完成率 | 判断营前准备完成度 |
| 入营评测完成率 | 判断学员入营准备 |
| 课程查看人数 | 判断课程触达 |
| 资料下载次数 | 判断资料使用情况 |
| 任务提交率 | 判断营地任务完成情况 |
| 创业想法提交数 | 判断核心成果产出 |
| 老师点评完成率 | 判断老师工作完成情况 |
| 家长查看次数 | 判断家长参与度 |
| 问题反馈数与处理时长 | 判断服务质量 |

### 5.4 运营需要的后台功能

| 运营任务 | 后台能力 |
| --- | --- |
| 开新营 | 营地/期次管理 |
| 管用户 | 老师、学员、家长账号管理 |
| 管关系 | 家长-学员绑定、老师-营地/队伍绑定 |
| 管内容 | 课程、资料、工作坊、任务管理 |
| 管成果 | 任务提交、创业想法、商业计划书 |
| 管团队 | 队伍、队长、队员、项目名称 |
| 管评价 | 老师点评、任务点评、评分 |
| 管相册 | 照片上传、分组、展示 |
| 管反馈 | 问题反馈、处理状态、回复记录 |
| 做复盘 | 数据统计、导出 |

### 5.5 运营风险

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| 配置链路太长 | 运营可能不知道先配什么 | 后台增加营地配置向导 |
| 人员关系错误 | 家长绑定错孩子、老师负责范围错 | 关键绑定关系支持确认和批量检查 |
| 内容发布不及时 | 学员看不到课程、任务、资料 | 支持发布状态、定时发布、预览 |
| 家长反馈无人处理 | 影响服务口碑 | 设置处理状态和负责人 |
| 点评不完整 | 家长看不到孩子反馈 | 老师点评完成率进入运营看板 |

## 6. 从交互设计角度梳理

### 6.1 整体交互原则

这个产品的交互要服务“高频查看 + 低频提交 + 多角色差异”。

设计原则：

- 首页工作台化：不同角色首页直接呈现最重要任务。
- 操作入口明确：任务提交、创业想法、队伍编辑、问题反馈要容易找到。
- 弹窗适合轻量操作：列表详情、资料下载、课程查看可以用弹窗。
- 长表单要分区：创业想法、问题反馈、注册补充信息不能堆在一起。
- 状态要可见：任务是否提交、反馈是否处理、资料是否完成，都要展示状态。
- 权限差异要可感知：队长/队员、老师/家长看到的入口不同。

### 6.2 信息架构建议

用户端不建议使用复杂导航，而应该使用角色工作台。

学员首页信息优先级：

1. 当前营地与身份。
2. 待完成任务。
3. 创业想法提交。
4. 我的课程。
5. 工作坊与资料。
6. 我的团队。
7. 营地相册。

老师首页信息优先级：

1. 学生搜索与筛选。
2. 学生列表。
3. 队伍设置。
4. 队伍成员管理。
5. 队员点评。
6. 资料下载。

家长首页信息优先级：

1. 选择营地和孩子。
2. 孩子点评。
3. 我的课程。
4. 云相册。
5. 问题反馈。

后台信息优先级：

1. 营地切换/租户选择。
2. 当前营地配置完成度。
3. 人员、内容、任务、团队、点评、反馈。
4. 数据统计与导出。

### 6.3 关键页面交互要求

登录注册：

- 注册第一步只收集最少信息：姓名、手机号、验证码、协议。
- 注册第二步再补充营地、身份证、地址、学生账号、密码。
- 多学生账号添加入口要明显，但默认只展示一个账号输入区域。

学员首页：

- 不要让学员在多个菜单里找任务，任务和创业想法应作为主入口。
- 队长身份要明确展示，并提供编辑队伍按钮。
- 队员身份不展示编辑按钮。

老师学生列表：

- 搜索框、营地筛选、学员筛选应固定在列表上方。
- 学生状态用标签呈现，如活跃、请假中、未报到。
- 查看详情按钮要统一。

队伍管理：

- 添加队员使用弹窗。
- 移除队员必须二次确认。
- 队伍成员列表应展示头像、姓名、角色。

点评：

- 老师点评和任务点评分开。
- 评分星级和文字点评都要保留。
- 可见性设置要清晰说明：公开给学生/家长，或仅管理员可见。

家长查看：

- 营地和孩子切换应在页面顶部。
- 点评内容要按时间和类型展示。
- 云相册图片应支持预览大图。

问题反馈：

- 类型、标题、描述、图片、联系方式、匿名提交。
- 提交后要给出成功反馈。
- 后台处理后建议有处理状态或回复。

### 6.4 组件规范建议

| 组件 | 使用场景 | 设计要求 |
| --- | --- | --- |
| 顶部栏 | 所有用户端页面 | 品牌、设置入口、登录态 |
| 用户卡片 | 三类用户首页 | 头像、姓名、角色标签 |
| 内容卡片 | 课程、资料、团队、相册 | 标题、更多入口、列表内容 |
| 表格 | 老师学生列表、后台管理 | 支持搜索、筛选、分页、状态标签 |
| 弹窗 | 列表详情、提交、编辑 | 标题居中、右上关闭、主按钮固定 |
| 上传组件 | 任务、反馈、计划书 | 图片预览、文件类型限制、大小提示 |
| 标签 | 队伍、状态、角色 | 颜色区分但不要过多 |
| Tab | 点评、队伍管理 | 当前项下划线或高亮 |

### 6.5 交互风险

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| 首页信息太多 | 学员、老师、家长首页都偏重 | 按角色精简第一屏重点 |
| 弹窗层级过深 | 设置、队伍、任务、相册都有弹窗 | 避免弹窗里再弹复杂弹窗 |
| 长表单压力大 | 创业想法提交内容多 | 分区、占位提示、保存草稿 |
| 状态不明显 | 用户不知道是否提交成功 | 增加状态标签和提交结果反馈 |
| 多角色共用样式但权限不同 | 容易误导用户 | 不可用功能不要展示，或明确置灰说明 |

## 7. MVP 范围建议

为了先完成可上线版本，建议 MVP 聚焦下面能力。

### 7.1 必须做

- 登录注册。
- 学员、老师、家长三类首页。
- 营地切换/选择。
- 学员课程、资料、工作坊、任务列表。
- 学员任务提交。
- 学员创业想法提交。
- 老师学生列表。
- 老师队伍管理。
- 老师点评。
- 家长查看课程、点评、相册。
- 家长问题反馈。
- 后台营地、用户、课程、资料、任务、队伍、点评、相册、反馈基础管理。
- 基础权限控制。

### 7.2 可以后置

- 复杂数据看板。
- 老师常用评语库。
- 点评草稿。
- 相册分组高级筛选。
- 消息通知。
- 入营评测自动评分。
- 移动端适配。
- 批量导入导出高级模板。

## 8. 需求优先级

| 优先级 | 需求 | 原因 |
| --- | --- | --- |
| P0 | 账号、角色、营地、权限 | 没有它无法区分用户和数据 |
| P0 | 学员任务提交、创业想法提交 | 核心营地成果 |
| P0 | 老师学生列表、队伍管理、点评 | 核心老师工作流 |
| P0 | 家长查看点评、相册、反馈 | 核心家长价值 |
| P0 | 后台营地/用户/内容/任务配置 | 用户端数据来源 |
| P1 | 入营资料填写流程 | 营前准备关键，但可按表单逐步完善 |
| P1 | 数据统计与完成率 | 运营复盘需要 |
| P1 | 批量导入导出 | 减轻运营压力 |
| P2 | 消息通知、草稿、模板 | 提升体验 |

## 9. 当前待确认问题

这些问题需要在正式开发前确认，否则会影响数据模型和权限设计。

1. 账号体系：家长账号和学生账号是独立登录，还是家长创建学生子账号？
2. 学员是否可以自己注册，还是必须由家长注册并配置？
3. 老师负责范围按营地、班级、队伍，还是单独绑定学生？
4. 队长是否可以自由添加队员，还是只能从老师/后台提供的候选名单中添加？
5. 创业想法提交后，是否需要老师点评、后台审核、家长可见？
6. 任务提交后是否允许修改？
7. 老师点评公开后，家长和学生是否都可见？
8. 家长是否能看到学生任务原文和创业想法全文？
9. 相册是否由后台上传，还是老师/学员也能上传？
10. 问题反馈是否需要后台回复并同步给家长？
11. 后台是否存在多租户，不同机构是否共用一套系统？
12. 是否需要支持移动端 H5 或小程序？
