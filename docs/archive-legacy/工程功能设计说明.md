# 少年马斯克创业营用户端 Design.md

## 1. 设计定位

用户端前端采用「蓝色科技感 + 青少年成长营地 + 轻量 SaaS 工作台」的视觉方向。

整体气质要像一个面向学生、家长、老师的营地小程序/移动 SaaS，而不是传统后台系统。界面需要有科技感、行动感、成长感，同时保持信息清晰、操作轻便。

关键词：

- 年轻、明亮、可信、专业
- 蓝色科技、未来感、创业营、成长陪伴
- 卡片化、轻插画、移动端优先
- 简洁 SaaS 框架，不做沉重管理后台

## 2. 参考图提炼

参考图中已经形成了较明确的视觉语言：

- Logo 使用高饱和蓝色渐变，品牌识别强，适合作为登录页、启动页、顶部栏主视觉。
- 登录背景是蓝色渐变叠加城市、天空、奔跑孩子插画，表达「奔向未来」「改变世界」。
- 功能入口卡片使用浅蓝底、深蓝标题、灰色说明文字、右侧插画或箭头。
- 图标多为蓝色主形，局部用黄色、紫色、青色做点缀。
- 卡片圆角偏大，整体柔和，不使用锋利、冰冷的企业后台风格。

因此用户端应避免纯白表格后台感，应该让首页、任务、想法、相册、反馈等页面更像「营地任务中心」和「成长记录中心」。

## 3. 设计原则

### 3.1 移动端优先

学生和家长端优先按手机宽度设计，桌面端只是自然扩展。核心内容以单列卡片、横向滑动模块、底部导航或紧凑顶部导航呈现。

老师端可以更偏工作台，但也要保留品牌蓝、浅色卡片和清晰模块，不做密集后台表格首页。

### 3.2 信息轻、行动明确

每个页面优先呈现 1 个主任务和 2 到 4 个次级入口。文案尽量短，按钮动词明确，例如「提交想法」「查看点评」「上传照片」「去完成」。

### 3.3 插画服务功能

插画和 3D 图形只用于登录页、空状态、入口卡片、活动模块，不要在表格、表单、审批类页面中过度装饰。

### 3.4 SaaS 可复用

所有视觉样式应沉淀为 token 和组件规则，避免每个页面单独写颜色、阴影、圆角。

## 4. 色彩系统

### 4.1 品牌主色

```css
:root {
  --brand-700: #003CAB;
  --brand-600: #0052D9;
  --brand-500: #0063EA;
  --brand-400: #2F80FF;
  --brand-300: #5B93F6;
  --brand-100: #EAF3FF;
  --brand-050: #F2F7FF;
}
```

使用规则：

- `--brand-600` 用于主按钮、顶部品牌文字、关键链接。
- `--brand-500` 用于大面积品牌栏、选中态、重要图标。
- `--brand-100` 和 `--brand-050` 用于功能入口卡片、页面浅蓝背景。
- 登录页可以使用 `#0052D9` 到 `#2F80FF` 或 `#003CAB` 的渐变。

### 4.2 辅助色

```css
:root {
  --accent-yellow: #FFC400;
  --accent-cyan: #35C8FF;
  --accent-purple: #7666FF;
  --success: #00A870;
  --warning: #ED7B2F;
  --danger: #D54941;
}
```

使用规则：

- 黄色用于任务完成、积分、徽章、重点提示的小面积点缀。
- 青色用于科技感、相册、上传、互动反馈。
- 紫色用于反馈、消息、AI 点评、创意灵感类模块。
- 辅助色面积不超过单屏视觉的 15%，保持蓝色为主。

### 4.3 中性色

```css
:root {
  --text-1: #181818;
  --text-2: #595959;
  --text-3: #8C8C8C;
  --text-inverse: #FFFFFF;
  --bg-page: #F5F7FA;
  --bg-soft: #F2F7FF;
  --bg-card: #FFFFFF;
  --border-light: #E7E7E7;
  --border-soft: #DDEBFF;
}
```

页面背景以 `#F5F7FA` 或非常浅的蓝灰为主。不要使用大面积纯深蓝背景，除非是登录页或品牌欢迎页。

## 5. 字体与排版

字体栈：

```css
font-family: "Noto Sans SC", "Noto Sans", -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif;
```

字号层级：

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 登录页主标题 | 28-32px | 700 | 1.35 |
| 页面标题 | 20-24px | 700 | 1.35 |
| 卡片标题 | 18-20px | 700 | 1.35 |
| 模块标题 | 16-18px | 600 | 1.4 |
| 正文 | 14-16px | 400 | 1.6 |
| 辅助说明 | 12-14px | 400 | 1.5 |

中文标题可以更粗，但不要使用负字距。按钮和标签文字保持 14-16px，避免过小。

## 6. 圆角、阴影、边框

```css
:root {
  --radius-xs: 6px;
  --radius-sm: 8px;
  --radius-md: 12px;
  --radius-lg: 16px;
  --radius-xl: 20px;
  --shadow-card: 0 8px 24px rgba(0, 82, 217, 0.08);
  --shadow-float: 0 12px 32px rgba(0, 82, 217, 0.14);
}
```

使用规则：

- 普通数据卡片：8px 或 12px。
- 功能入口卡片：16px。
- 登录面板、底部弹窗、重点模块：16px 到 20px。
- 阴影使用蓝色透明阴影，不使用厚重黑色阴影。
- 卡片边框优先使用浅蓝边框或无边框，避免灰线过多。

## 7. 页面结构

### 7.1 登录页

登录页是品牌第一印象，应保留参考图的蓝色插画氛围。

建议结构：

- 移动端：全屏蓝色插画背景，上方 Logo，中部价值主张，下方白色或半透明登录面板。
- 桌面端：居中登录面板，背景可使用蓝色渐变和城市/成长插画，不做普通后台登录框。
- 主文案方向：「让每个孩子都有改变世界的潜力」。
- 登录按钮使用品牌蓝渐变。

视觉规则：

- 背景允许更强视觉表现。
- 表单区域必须清晰，输入框对比足够。
- 不要把登录页做成纯白卡片加一行标题。

### 7.2 学生端

学生端重点是任务推进和创意表达，风格可以最活泼。

首页建议模块：

- 今日任务
- 我的创业想法
- 团队进度
- 课程材料
- 营地相册

视觉规则：

- 使用浅蓝功能卡片。
- 关键入口可配图标或插画。
- 任务状态用进度条、徽章、完成标记呈现。
- 文案更鼓励式，例如「继续完善你的创意」。

### 7.3 家长端

家长端重点是安心、透明、陪伴。

首页建议模块：

- 孩子今日动态
- 课程进度
- 老师点评
- 营地相册
- 问题反馈

视觉规则：

- 比学生端更克制，减少强动效。
- 用浅蓝、白色、少量紫色反馈插画。
- 信息卡片强调「状态」「时间」「老师反馈」。
- 避免太游戏化，保持可信赖。

### 7.4 老师端

老师端重点是效率和批量处理，但仍属于用户端，不应完全后台化。

首页建议模块：

- 今日待点评
- 学员提交动态
- 营地记录
- 课程材料
- 班级/营地概览

视觉规则：

- 可以使用更规整的网格和列表。
- 表格页面保留 SaaS 效率，但顶部和空状态保持品牌视觉。
- 批量操作按钮要明确，不要过度装饰。

## 8. 组件规范

### 8.1 顶部导航

当前代码已有白底顶部栏和蓝色品牌文字，可升级为：

- 桌面端：白底、底部浅边框、品牌 Logo + 角色导航。
- 移动端：顶部 Logo + 当前页面标题，主要导航放到底部 Tab 或抽屉。
- 顶部栏高度：桌面 64-70px，移动 48-56px。
- 品牌区域使用 Logo 图，不只用文字。

### 8.2 底部导航

学生端和家长端建议使用底部 Tab：

- 学生端：首页、任务、想法、团队、我的。
- 家长端：首页、课程、相册、点评、反馈。
- 选中态使用品牌蓝图标和文字。
- 未选中态使用 `--text-3`。

### 8.3 功能入口卡片

参考「提交创业想法」「营地相册」「问题反馈」。

尺寸建议：

- 横向大卡：高度 96-120px，圆角 16px，内边距 20px。
- 方形/半宽卡：高度 150-180px，圆角 16px，内边距 20px。

视觉规则：

- 背景：`--brand-100` 或 `#EAF3FF`。
- 标题：品牌蓝，18-22px，700。
- 描述：灰色，14-15px。
- 右侧可放插画、图标或箭头。
- 卡片点击态：轻微上浮 2px，阴影增强。

### 8.4 普通信息卡片

- 背景白色。
- 圆角 12px。
- 内边距 16px 或 20px。
- 标题 16px/600。
- 用浅边框或浅阴影分组。
- 不要在卡片里再套卡片。

### 8.5 按钮

主按钮：

```css
background: linear-gradient(135deg, #0063EA 0%, #2F80FF 100%);
color: #fff;
border-radius: 10px;
height: 44px;
font-weight: 600;
```

次按钮：

- 白底蓝字，浅蓝边框。
- Hover 使用浅蓝背景。

危险按钮：

- 只在删除、撤销提交等场景出现。
- 不要与主按钮并列时抢视觉。

### 8.6 标签与状态

状态标签建议：

- 未开始：灰色
- 进行中：蓝色
- 待点评：黄色/橙色
- 已完成：绿色
- 需修改：红色

标签圆角使用 999px，字号 12px，内边距 4px 8px。

### 8.7 空状态

空状态不要只显示默认组件文字。建议使用轻插画 + 一句话 + 一个行动按钮。

示例：

- 空想法：「还没有创业想法，先记录一个小灵感吧」
- 空相册：「营地精彩瞬间即将出现」
- 空点评：「老师点评完成后会显示在这里」

## 9. 图标与插画

图标方向：

- 扁平蓝色图标，允许黄色局部底边或紫色点缀。
- 图标线条和块面都可以，但要保持圆润、简洁。
- 图标容器可使用蓝色渐变圆角方块。

插画方向：

- 登录页使用完整场景插画。
- 功能入口使用小型 3D/轻拟物插画。
- 角色头像可以使用圆润插画头像，不使用写实照片作为默认头像。

不要使用：

- 黑白线性后台图标作为主视觉。
- 过度卡通、低幼风格。
- 与品牌无关的通用 stock 图片。

## 10. 动效

动效应该轻，不打扰学习和工作。

推荐：

- 卡片 hover 上浮：`translateY(-2px)`。
- 页面模块进入：轻微 fade-in-up，时长 200-300ms。
- 按钮点击：按压缩放到 0.98。
- 进度条变化：300ms ease-out。

避免：

- 大面积循环动画。
- 弹跳过强的游戏化动效。
- 影响表单输入和老师批量操作的动画。

## 11. 响应式规则

断点建议：

```css
--screen-sm: 640px;
--screen-md: 768px;
--screen-lg: 1024px;
--screen-xl: 1200px;
```

移动端：

- 页面左右边距 16px。
- 卡片单列。
- 功能入口优先竖向排列。
- 底部 Tab 固定。

平板：

- 页面左右边距 24px。
- 卡片可 2 列。

桌面：

- 内容最大宽度 1200px。
- 学生/家长端仍保持轻量卡片。
- 老师端可使用 2 到 4 列概览 + 列表。

## 12. 与当前技术栈的落地建议

当前用户端使用 Vue + TDesign，已有全局变量：

```css
--primary:#0052D9;
--bg:#F5F7FA;
--card:#FFF;
--border:#E7E7E7;
```

建议下一步将全局样式升级为完整 token：

```css
:root {
  --primary: #0052D9;
  --primary-hover: #0063EA;
  --primary-light: #EAF3FF;
  --primary-soft: #F2F7FF;
  --text1: #181818;
  --text2: #595959;
  --text3: #8C8C8C;
  --bg: #F5F7FA;
  --card: #FFFFFF;
  --border: #E7E7E7;
  --border-blue: #DDEBFF;
  --radius-card: 12px;
  --radius-feature: 16px;
  --shadow-card: 0 8px 24px rgba(0, 82, 217, 0.08);
}
```

同时建议新增基础类：

- `.app-page`
- `.app-section`
- `.app-card`
- `.feature-card`
- `.status-tag`
- `.page-title`
- `.muted-text`

## 13. 设计禁止项

- 不要把用户端做成后台管理系统。
- 不要大量使用纯表格作为首页。
- 不要大面积使用深蓝或紫蓝渐变，登录页除外。
- 不要使用过大的圆角胶囊按钮堆满页面。
- 不要在卡片中嵌套卡片。
- 不要让插画遮挡正文或按钮。
- 不要用灰色空状态结束页面，必须提供下一步行动。
- 不要让学生端、家长端、老师端完全长得一样，应共享品牌但区分重心。

## 14. 质量检查清单

每个用户端页面交付前检查：

- 是否使用品牌 Logo 或品牌蓝建立识别。
- 是否移动端优先，375px 宽度下无文字溢出。
- 首页是否有明确的主行动。
- 卡片圆角、阴影、间距是否统一。
- 主按钮是否只有一个最强视觉。
- 空状态是否有说明和行动。
- 学生端是否更有活力，家长端是否更安心，老师端是否更高效。
- 色彩是否以蓝色为主，辅助色只做点缀。
- 插画是否服务功能，而不是纯装饰。
- 是否复用了全局 token，而不是散落硬编码。

## 15. 一句话风格定义

少年马斯克创业营用户端是一个明亮、可信、带有未来感的蓝色成长型 SaaS：学生看到任务和创造力，家长看到陪伴和反馈，老师看到清晰高效的营地工作台。
