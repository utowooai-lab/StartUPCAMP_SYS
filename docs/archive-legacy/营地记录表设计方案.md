# 少年马斯克创业营营地记录表设计方案

版本：v0.1  
日期：2026-06-02  
用途：老师记录单个学员 6 天 5 夜营地表现，并作为 AI 点评与一键报告生成的数据来源。

## 1. 设计目标

营地记录表不是普通签到表，也不是单纯评语表。它需要同时满足三件事：

1. 帮老师在营期中快速记录单个学员的关键表现。
2. 按 6 天 5 夜课程流程沉淀可回溯证据。
3. 为 AI 生成点评和成长报告提供结构化素材。

最终输出应支持：

- 老师端：每日记录、关键事件、测评维度、AI 点评建议。
- 后台端：查看学员记录、统计记录完整度、生成/审核报告。
- 学员/家长端：查看经过老师审核发布后的成长报告。

## 2. 现有反馈 PDF 可复用维度

从学员反馈 PDF 中提取到的报告维度如下：

| 一级维度 | 二级维度 | 说明 |
| --- | --- | --- |
| 知识 Knowledge | 词汇学习、知识点、理论 | 对创业相关概念、理论、词汇的理解与应用 |
| 能力技能 Competence & Skills | 学习力、人际沟通能力、解决问题能力、团队合作、理解力 | 学习、表达、协作、问题处理能力 |
| 思维养成 Thinking | 创新思维、创业思维、批判性思维、设计思维 | 创业营核心思维训练结果 |
| 精神素养 Quality Intelligence | 自信、分享坚毅、同理心、合作精神 | 情绪、心态、人际和团队品质 |
| 担当 Well-being | 全球视野、使命感、责任感 | 责任意识、社会视角、目标感 |
| 探究与创新 Deep Thinking & Innovation | 坚持、深度思考、与众不同 | 面对问题时的持续探究与独特表达 |
| 课程状态 Course Status | 参与情况 | 课堂参与、互动、专注度 |
| 自理能力 Self Care Ability | 自理阶段 | 6 天 5 夜生活场景下的独立生活能力 |

建议保留这些维度作为报告输出维度，但老师日常记录不应一次填写全部维度，否则负担过重。日常记录应围绕当天课程流程，自动映射到这些维度。

## 3. 营地 6 天 5 夜流程拆解

根据当前流程图，营期主线如下：

| 天数 | 核心主题 | 关键活动 | 主要观察方向 |
| --- | --- | --- | --- |
| 第一天 | 接营与适应 | 接营、破冰、生活适应 | 安全感、自理能力、适应力、社交初始状态 |
| 第二天 | 激发梦想与组队 | 开营仪式、优化 Idea Pitch、个人路演、小组会议 | 表达意愿、兴趣方向、团队角色、主动性 |
| 第三天 | 问题发现与定义 | 围绕项目发现问题、定义问题、头脑风暴、市场调研 | 问题意识、理解力、调研能力、解决问题能力 |
| 第四天 | 用户调研与 MVP | 用户调研、产品 MVP 构思、宣传方案、MVP 制作 | 用户意识、设计思维、动手能力、团队协作 |
| 第五天 | 商业路演准备 | 商业路演培训、PPT 制作、演讲稿、工作时间 | 表达能力、逻辑组织、坚持、责任感 |
| 第六天 | 路演与结营 | 路演、颁奖、结营仪式 | 自信展示、综合成长、项目表达、反思总结 |

## 4. 营地记录表总体结构

建议把营地记录拆成 4 张表：

1. **每日学员观察记录表**：老师按天记录学员表现。
2. **关键事件记录表**：记录特别值得进入报告的高光/困难/转折事件。
3. **能力维度测评表**：按维度做 A/B/C 等级判断。
4. **AI 报告素材确认表**：生成报告前确认哪些记录进入报告。

这样可以兼顾老师填写效率和报告质量。

## 5. 表一：每日学员观察记录表

### 5.1 使用场景

每天至少记录 1 次，针对单个学员。老师可以在手机端快速填写，也可以在后台补录。

### 5.2 字段设计

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 营地 | 选择 | 是 | 例如第 27 期创业营 |
| 学员 | 选择 | 是 | 单个学员 |
| 记录日期 | 日期 | 是 | 第几天对应自然日期 |
| 营地天数 | 枚举 | 是 | 第一天至第六天 |
| 记录老师 | 自动 | 是 | 当前登录老师 |
| 观察时段 | 多选 | 是 | 第一节、团建、第二节、午休、第三节、活动、第四节、晚餐、第五节 |
| 课程/活动主题 | 选择 | 是 | 根据当天流程自动带出，可手动调整 |
| 学员参与状态 | 枚举 | 是 | 主动参与、适当参与、被动参与、未参与 |
| 今日关键表现 | 长文本 | 是 | 记录具体行为，不写空泛评价 |
| 今日困难/挑战 | 长文本 | 否 | 学员遇到的问题、卡点、情绪变化 |
| 老师介入方式 | 多选 | 否 | 提问引导、情绪安抚、任务拆解、示范、同伴协助、未介入 |
| 学员回应 | 长文本 | 否 | 学员在老师介入后的变化 |
| 证据素材 | 图片/附件 | 否 | 照片、作品、任务截图、路演截图等 |
| 关联维度 | 多选 | 是 | 从能力维度中选择 1-5 个 |
| 今日一句话总结 | 文本 | 是 | 便于 AI 报告引用 |
| 是否推荐进入报告 | 布尔 | 是 | 默认否，老师可勾选 |

### 5.3 填写示例

| 字段 | 示例 |
| --- | --- |
| 营地天数 | 第三天 |
| 观察时段 | 第三节、第四节 |
| 课程/活动主题 | 头脑风暴、市场调研 |
| 学员参与状态 | 适当参与 |
| 今日关键表现 | 在头脑风暴中一开始不敢主动发言，但听到队友讨论“书包太重”后，用画画方式提出“长翅膀的书包”方案。 |
| 今日困难/挑战 | 文字表达和书写速度较慢，影响想法输出。 |
| 老师介入方式 | 提问引导、任务拆解 |
| 学员回应 | 在老师建议用图画代替文字后，能更流畅地表达创意。 |
| 关联维度 | 创新思维、设计思维、自信、表达能力 |
| 今日一句话总结 | 她通过画画找到了适合自己的表达方式，展现出丰富想象力。 |
| 是否推荐进入报告 | 是 |

## 6. 表二：关键事件记录表

### 6.1 使用场景

用于记录特别值得写进家长报告的事件。不是每天必填，而是在出现明显成长、高光、困难、转折时填写。

### 6.2 字段设计

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 学员 | 选择 | 是 | 单个学员 |
| 事件日期 | 日期 | 是 | 事件发生日期 |
| 营地天数 | 枚举 | 是 | 第一天至第六天 |
| 事件类型 | 枚举 | 是 | 高光表现、困难突破、团队协作、主动担当、创意表达、情绪变化、生活自理 |
| 事件标题 | 文本 | 是 | 10-30 字概括 |
| 事件背景 | 长文本 | 是 | 当时发生了什么 |
| 学员行为 | 长文本 | 是 | 学员具体做了什么，说了什么 |
| 结果影响 | 长文本 | 是 | 对自己、团队或项目产生了什么影响 |
| 老师解读 | 长文本 | 是 | 老师如何理解这个行为背后的能力或品质 |
| 可关联报告段落 | 多选 | 是 | 欣赏点、成长建议、祝福、能力评价 |
| 证据素材 | 图片/附件 | 否 | 作品、照片、视频截图 |
| AI 引用优先级 | 枚举 | 是 | 高、中、低 |

### 6.3 事件类型说明

| 事件类型 | 适合记录的内容 |
| --- | --- |
| 高光表现 | 学员主动表达、完成展示、提出好创意 |
| 困难突破 | 从不敢做到愿意做，从不会到学会 |
| 团队协作 | 配合队友、承担角色、解决分歧 |
| 主动担当 | 主动承担任务、服务团队、对结果负责 |
| 创意表达 | 独特想法、产品方案、故事表达、绘画表达 |
| 情绪变化 | 从紧张到放松，从失落到重新参与 |
| 生活自理 | 整理物品、用餐、作息、个人管理 |

## 7. 表三：能力维度测评表

### 7.1 使用场景

建议每天可选填，营期结束必须完成。用于生成报告中的等级和能力分析。

### 7.2 等级定义

沿用现有反馈报告中的三阶段：

| 等级 | 名称 | 含义 |
| --- | --- | --- |
| A | 领导与创新 | 能主动领导、深入分析、独立完成、创新应用 |
| B | 探索与实践 | 能理解并应用，能在团队中承担角色并完成任务 |
| C | 启蒙与发现 | 在老师引导下理解、参与、尝试和完成基础任务 |
| N | 未观察到 | 当天或本营期没有足够证据判断 |

### 7.3 维度字段

| 一级维度 | 二级维度 | 评分 | 证据要求 |
| --- | --- | --- | --- |
| 知识 | 词汇学习 | A/B/C/N | 需要关联课程表达、任务回答或路演内容 |
| 知识 | 知识点 | A/B/C/N | 需要关联商业知识应用场景 |
| 知识 | 理论 | A/B/C/N | 需要关联问题定义、市场调研、商业模式等 |
| 能力技能 | 学习力 | A/B/C/N | 需要关联学习任务或老师引导后的变化 |
| 能力技能 | 人际沟通能力 | A/B/C/N | 需要关联团队讨论、采访、路演沟通 |
| 能力技能 | 解决问题能力 | A/B/C/N | 需要关联问题发现、方案设计、冲突处理 |
| 能力技能 | 团队合作 | A/B/C/N | 需要关联组队、任务分工、小组会议 |
| 能力技能 | 理解力 | A/B/C/N | 需要关联课程理解、任务理解、反馈吸收 |
| 思维养成 | 创新思维 | A/B/C/N | 需要关联新想法、新方案 |
| 思维养成 | 创业思维 | A/B/C/N | 需要关联商业机会、用户需求、价值创造 |
| 思维养成 | 批判性思维 | A/B/C/N | 需要关联质疑、比较、判断、取舍 |
| 思维养成 | 设计思维 | A/B/C/N | 需要关联用户、原型、测试、迭代 |
| 精神素养 | 自信 | A/B/C/N | 需要关联公开表达或主动尝试 |
| 精神素养 | 分享坚毅 | A/B/C/N | 需要关联分享、坚持、面对困难 |
| 精神素养 | 同理心 | A/B/C/N | 需要关联理解他人需求或情绪 |
| 精神素养 | 合作精神 | A/B/C/N | 需要关联协商、配合、共识 |
| 担当 | 全球视野 | A/B/C/N | 需要关联国际视角、社会议题 |
| 担当 | 使命感 | A/B/C/N | 需要关联目标、责任、价值感 |
| 担当 | 责任感 | A/B/C/N | 需要关联任务承担、后果意识 |
| 探究与创新 | 坚持 | A/B/C/N | 需要关联持续尝试、遇挫不放弃 |

| 课程状态 | 参与情况 | A/B/C/N | 每日建议必填 |
| 自理能力 | 自理能力 | A/B/C/N | 生活老师或带队老师记录 |

### 7.4 单项测评字段

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 维度 | 自动/选择 | 是 | 如创新思维 |
| 等级 | A/B/C/N | 是 | 选择当前判断 |
| 证据说明 | 长文本 | A/B/C 必填 | 必须写具体行为 |
| 关联记录 | 多选 | 否 | 关联每日记录或关键事件 |
| 老师建议 | 长文本 | 否 | 给学员或家长的建议 |

## 8. 表四：AI 报告素材确认表

### 8.1 使用场景

营期结束前，老师生成报告前使用。系统先自动汇总，老师确认后再生成 AI 报告。

### 8.2 字段设计

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 学员 | 选择 | 是 | 单个学员 |
| 报告周期 | 日期范围 | 是 | 本期营地 6 天 5 夜 |
| 基础信息 | 自动 | 是 | 来自档案收集表/信息收集表 |
| 创业梦想/问题 | 自动/编辑 | 是 | 来自信息收集或营地任务 |
| 项目主题 | 自动/编辑 | 是 | 学员或队伍项目 |
| 高光事件 | 多选 | 是 | 至少 3 条 |
| 成长挑战 | 多选 | 是 | 至少 1 条 |
| 能力等级 | 自动 | 是 | 来自能力维度测评 |
| 老师补充观察 | 长文本 | 否 | 给 AI 的额外素材 |
| 家庭建议方向 | 多选 | 是 | 表达自信、创意表达、团队协作、自理能力、学习习惯等 |
| 报告语气 | 枚举 | 是 | 温暖鼓励、专业客观、活泼亲切 |
| 生成范围 | 多选 | 是 | 欣赏点、能力评价、成长建议、祝福语 |

## 9. 按 6 天流程设计的记录模板

### 9.1 第一天：接营与适应

| 记录重点 | 推荐维度 | 老师可观察问题 |

| 记录重点 | 推荐维度 | 老师可观察问题 |

| 结营反思 | 使命感、深度思考 | 是否能说出自己的成长和收获？ |
| 综合表现 | 与众不同、坚持、责任感 | 本期最值得被看见的变化是什么？ |

## 10. 老师端填写交互建议

### 10.1 每日快速记录

老师端首页增加“今日记录”入口：

- 选择学员。
- 选择当天活动。
- 选择参与状态。
- 写 1 段具体表现。
- 勾选关联维度。
- 选择是否进入报告。

目标：单条记录 2 分钟内完成。

### 10.2 深度记录

当出现高光或转折事件时，老师进入“关键事件记录”：

- 写清楚背景、行为、影响、老师解读。
- 标记 AI 引用优先级。
- 上传素材。

目标：单条记录 5 分钟内完成。

### 10.3 营期末测评

营期结束前，系统提示老师完成：

- 缺失维度检查。
- 每个学员至少 3 条高光事件。
- 每个学员至少 1 条成长建议素材。
- 每个学员参与情况和自理能力必填。

### 11.3 字段分层建议

为了减少老师和 AI 使用时的混乱，建议后台把字段分成以下数据层：

| 数据层 | 包含字段 | 使用者 | 说明 |
| --- | --- | --- | --- |
| 身份与安全信息 | 姓名、身份证、性别、监护人、电话、地址、紧急联系人 | 后台、老师 | 只用于管理和安全，不进入家长报告正文 |
| 学校与经历背景 | 学校、语言、去过国家、过往营地 | 老师、AI | 用于理解孩子背景，避免泛泛评价 |
| 家长期待 | 教育问题、家长眼中的孩子、孩子能带来的价值 | 老师、AI | 用于老师关注重点和家庭建议 |
| 学员自我表达 | 好奇的问题、英雄/偶像、成就感、户外运动、希望收获 | 老师、AI | 用于营地记录观察方向和报告个性化 |
| 合规确认 | 家长确认与协议 | 后台 | 用于合规留档 |

### 11.4 与营地记录的关联方式

成长档案字段应在老师做营地记录时提供“观察提示”，但不要求老师重复填写。

| 表单信息 | 记录表提示方式 | 示例 |
| --- | --- | --- |
| 家长提出的教育问题 | 老师记录时展示“家长期待关注点” | 家长关注孩子是否敢于表达，老师记录时优先观察表达主动性 |
| 家长眼中的孩子 | 作为入营前画像 | 家长认为孩子内向，报告可对比营期中公开表达的变化 |
| 孩子最好奇/想解决的问题 | 自动关联创业问题线索 | 如果孩子写“书包太重”，可关联创业想法和问题发现 |
| 英雄/偶像 | 用于价值观和使命感分析 | 孩子崇拜科学家，可在报告中关联探索精神 |
| 最有成就感的事 | 用于自信和坚持维度 | 结合营地路演表现分析自信成长 |
| 希望创业营带来什么 | 用于结营报告目标回看 | 报告结尾对照“入营期待”和“实际收获” |

### 11.5 对 AI 报告的约束

这些表单字段包含身份证、住址、手机号等隐私信息，AI 报告生成时必须做字段级过滤：

- 不允许进入报告正文：身份证号、家庭住址、监护人手机号、紧急联系人手机号。
- 可作为背景但不直接评价：学校、去过国家、语言。
- 可用于个性化报告：教育问题、家长眼中的孩子、好奇的问题、英雄/偶像、成就感、希望收获。
- 报告中引用家长或学员原话时，需要经过老师确认。

### 11.9 输出结构

建议 AI 报告结构：

1. 学员基本信息。
2. 本期项目/创业梦想。
3. 营期高光表现。
4. 能力维度分析。
5. 成长挑战与建议。
6. 家庭陪伴建议。
7. 老师祝福。

### 11.10 生成规则

- AI 只能使用已确认的记录和素材，不凭空编造事件。
- 每条欣赏点必须关联至少一条老师记录。
- 能力评价必须引用测评等级和证据说明。
- 家庭建议必须结合成长挑战，不写泛泛建议。
- 报告发布前必须由老师或后台审核。
